# ADR 2026-04-21-15 · 用户管理 UI（G10 / FU-6 补）

## Context

G9 真登录落地后，IAM UsersTab 还是只能看；管理员要改角色、删用户、重置密码必须 curl；自助改密没入口。

## Decision

- 3 个新后端端点全部复用 G9 的 enforceAcl(ADMIN) 门；不新建 router
- UsersTab 前端仅对"真 DB 行"（source='seed' 且 user_id 是正整数）启用操作按钮；session 行不可编辑
- 禁"改自己 roles" + "删自己"——两道保险，防管理员自锁
- Reset password 是"管理员强制设他人新密"，**与 self POST /password 互补**（后者需要旧密）
- ChangePasswordModal 作为独立组件；UserArea DEV 态时禁用（DEV BYPASS 没真用户）

## Consequences

**正面**
- 登录链路闭环；DEV 以外的真部署能完整管理用户
- 不新增 API 层 namespace（复用 /api/auth）

**负面 / 取舍**
- seed 与 db 用 source 字段区分，UsersTab 判断逻辑靠 `source='seed' && id 正整数`——将来 /users 返回结构若变这里要同步改
- 没做软删 —— 硬删立即丢 audit_log 里的 principal_user_id 反查，接受
- 没做邮箱冲突前置校验 —— 靠后端 409，UI 展 msg
