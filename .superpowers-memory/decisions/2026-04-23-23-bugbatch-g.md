## 设计决策

### D-001 · 为什么 `notranslate` 放在 HTML 层而不是组件层

选项：
- (a) `<meta name="google" content="notranslate">`（全站生效 / 一行）
- (b) `<html translate="no">`（标准 HTML5 属性 / 更通用）
- (c) 每个含中文的组件加 `translate="no"`（精细但几千个点）

**选 a+b 组合**：`<html lang="zh-CN" translate="no">` + meta。Chrome 会尊重 `notranslate` meta + `lang="zh-CN"` 的声明；`translate="no"` 属性是 HTML5 标准，其它浏览器翻译插件也识别。成本 = 2 行改 index.html。拒绝 (c)：维护成本不接受。

### D-002 · TBD-09 为什么从 useEffect 改 useQuery，而不是修手写代码

原代码看似正确（`let cancelled = false` + cleanup `cancelled = true`），但：
1. StrictMode 下首次 mount 的响应永远被吞 —— 依赖 interval 的第一次（30s 后）才能拿到 data，那 30s 窗口永远"加载中…"
2. 若响应 payload shape 错（比如代理 miss 吞成 HTML），静默失败不报错
3. 项目其它模块（Overview/Iam/Notebooks）全部用 `useQuery`，这里是唯一手写异步的孤岛

改 useQuery 同时解决 1+2+3，代码反而变短。D-003 兜底的 shape 校验也顺手加上。

### D-003 · /api/mcp/stats 的 shape 校验该在哪里做

三个位置：
- **前端 queryFn 里 throw**（本轮选项）：快速防御，不依赖后端变更
- 后端路由加 schema（Zod）校验：更正确但跨 change
- type 保证（编译期）：TypeScript 编译期不管运行时

选前端 throw：一行代码、立刻见效、错误有明确文案上 UI；**后端 Zod 校验列 Follow-up**（`mcp-stats-schema` 独立 change）。

### D-004 · BUG-18 确认不改代码的证据链

- `Layout.tsx:77` 字面是 `›`（U+203A），不是 `>>`
- 测试报告说 `/assets /iam /notebooks /spaces /eval` 都显示 `›`，只有 `/overview /agent` 显示 `>>`
- 这两个页面**没有自己的面包屑组件**，走的是同一个 Layout header
- Chrome 自动翻译开启 + `<html lang="en">` 会对部分特殊 Unicode 字符做启发式替换；已知 Google Translate 在中日韩翻译时会把 `›` rewrite 成 `>>`（Unicode ASCII-fallback 路径）
- TBD-02 防御生效后，/overview /agent 也会回到 `›`
