# ADR 2026-04-23-18 · Bug Batch A（小改 8 条）

## Context

`2026/4/23` 的自动化测试报告给了 24 条 bug。按根因分了 A~F 六批；本 ADR 专门记批 A —— 纯数据绑定 / 小文案 / 导航的快速修复，每条 < 10 行改动，不值得上 OpenSpec 契约。

工作流：C `superpowers-feature-workflow`（独立 UI 细节，无 OpenSpec）

## Decision · 8 条修复

| Bug | 位置 | 根因 | 修复 |
|---|---|---|---|
| BUG-03 笔记本列表页标题显示"的" | `apps/web/src/knowledge/Notebooks/index.tsx` | `NotebookCard` 渲染 `{nb.name}` 无兜底，DB 里 name 为空 / 空白时直接渲染 | 抽 `displayName(nb)` helper：trim 后为空则显示 `（未命名 · #<id>）` |
| BUG-04 详情页永久"加载中..." | `apps/web/src/knowledge/Notebooks/Detail.tsx:63` | `{notebook?.name ?? '加载中…'}` 没有区分 loading / error / empty-name 三种状态 | 改三分支：`err` → "加载失败"；`notebook === null` → "加载中…"；`name` 为空 → "（未命名笔记本 · #id）" |
| BUG-15 NaN 时间 | `Notebooks/index.tsx:130, 186` | `new Date(nb.updated_at_ms)` 当 `updated_at_ms` 为 null/undefined 时是 Invalid Date，`.getMonth()` 返 NaN，模板字符串输出 "NaN/NaN NaN:NaN" | 加 `hasValidTime` 守护：无效时间显示 `—` |
| BUG-18 面包屑分隔符不一致 | `Assets/Detail.tsx:50, 52` | Layout 的 topbar 用 `›`，Assets/Detail 自画的面包屑用 ` / ` | 统一到 `›`（两处） |
| BUG-19 Agent 页无面包屑 + 导航不高亮 | `apps/web/src/components/Layout.tsx:7` | `navItems` 数组漏了 `/agent` 条目 | 在 `/qa` 之后插入 `{ to: '/agent', label: 'Agent', icon: '🤖' }` |
| BUG-20 "权限…" 按钮文字截断感 | `Assets/Detail.tsx` 顶栏 | `🔒 权限…` 的 `…`（省略号字符）视觉上像被截断；按钮无 nowrap 也可能被父 flex 挤换行 | 文案改 `🔒 权限设置` + `whiteSpace: 'nowrap'` |
| BUG-23 入库说明含内部技术字段 | `apps/web/src/knowledge/SpaceTree/index.tsx:38` | page-sub 写着"与 /ingest 共享 metadata_asset 表"，是给开发者看的 | 改成"数据源 → 资产 · 树形浏览与内容预览" |
| BUG-24 eval 提示含开发路径 | `Eval/index.tsx:88` | hint 里引用 `eval/golden-set.template.jsonl` 文件路径 | 改成"内置有 10 题入门模板可直接选用" |

## Consequences

**正面**
- 8 条低风险 bug 全部收口，用户 UI 体感立刻变干净
- BUG-03/04/15 都做了"数据缺失兜底"，不只是修掉这一次的现象，同时防御未来其它 name 为空 / 时间戳缺失场景
- Agent 页首次具备与其它页一致的 header 面包屑 + 侧栏高亮，减少"哪个页面我在"的认知负担
- 所有文案从"技术术语 / 文件路径"改为"用户语言"

**负面 / 取舍**
- Layout 的 `navItems` 增加一项后，小窗口下 sidebar 可能更挤；暂未做折叠处理（观察几天再定）
- BUG-03 的根因数据是用户某个被错误创建的 notebook name 为空；本修复不清理脏数据，只做视觉兜底。如果 DB 里还有更多空 name notebook，用户可能主动找 admin 清理
- BUG-18 只统一了 Assets/Detail 一处，其它页如果将来有自画面包屑还得同步收敛（目前全站只有 Assets/Detail 自画，Layout 负责所有其它）
