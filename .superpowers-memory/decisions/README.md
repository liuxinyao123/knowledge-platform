# Architecture Decision Records (ADR)

按日期 + 序号命名，**永不覆盖**：

```
YYYY-MM-DD-<seq>-<slug>.md
例如：2026-04-21-01-rbac-jwt.md
```

每条 ADR 建议包含：

1. **Context**：做这个决定的背景与压力。
2. **Decision**：实际决定了什么。
3. **Consequences**：随之带来的取舍（正 / 负）。
4. **Links**：关联的 OpenSpec change、PR、看板卡。

若后续要推翻某条决定，**不要改老文件**——新建一条 ADR 引用旧编号并解释推翻原因。
